# WorkBuddy 自动化怎么创建：定时任务的安全入门

> 验证状态：B 级来源核对。本文依据官方 Automation 文档整理，尚未完成本项目的完整人工实测。入口名称、通知平台和调度能力可能随版本变化。

WorkBuddy 自动化可以在指定时间运行一次或重复运行，适合日报、周报、数据监控和周期性文件处理。但自动化会在你没有盯着屏幕时执行，因此必须比普通任务更严格地限制输入、输出和权限。

## 不要把第一次执行直接设成自动化

正确顺序：

1. 手动执行任务；
2. 检查结果；
3. 修正任务描述；
4. 再手动执行一次；
5. 确认输入路径和输出文件稳定；
6. 最后创建自动化。

发送、删除、覆盖、支付和公开发布类任务不适合未经人工确认直接自动运行。

## 官方支持的调度类型

官方 Automation 文档列出了：

- 每隔若干小时；
- 每天固定时间；
- 每周指定日期；
- 只执行一次。

实际可选项以当前客户端为准。

## 创建自动化的通用步骤

1. 从侧边栏打开 Automation；
2. 创建新的自动化任务；
3. 写清楚要完成的工作；
4. 设置执行频率和时间；
5. 选择需要的工作目录；
6. 保存并启用；
7. 第一次运行后人工检查结果。

## 推荐任务模板

```text
任务：生成每日销售数据检查报告。

输入：
- 只读取当前工作目录 input/ 中最新的 sales-YYYY-MM-DD.xlsx。

处理：
1. 检查空值、重复项和明显异常值；
2. 汇总销售额、订单量和退款量；
3. 与上一日数据比较；
4. 无法确认的异常标记“待确认”。

输出：
- 保存到 output/YYYY-MM-DD-sales-check.md；
- 不得覆盖已有文件；
- 不得修改 input/ 中的文件；
- 不得发送或公开发布；
- 没有符合命名规则的输入文件时，只生成失败说明。
```

## 自动化必须写清楚的五件事

### 1. 输入范围

不要写“检查我的文件”。应写具体目录、文件名规则和时间范围。

### 2. 输出位置

按日期生成新文件，避免每天覆盖同一份结果。

### 3. 无数据时怎么处理

明确要求没有输入时停止，并生成说明，不要用旧数据冒充新数据。

### 4. 失败时怎么处理

要求记录失败步骤、错误提示和已生成文件，不要无限重试。

### 5. 哪些动作禁止自动执行

默认禁止发送、删除、覆盖、付费、账号修改和公开发布。

## 通知功能

官方文档提到，配置 Assistant 和对应消息平台后，自动化结果可以推送通知。是否支持以及支持哪些平台应以当前账号和版本为准。

即使启用通知，也建议只推送摘要和结果链接，不要把含隐私或业务数据的完整文件自动发送到群聊。

## 如何暂停和关闭

建立自动化后，记录：

- 自动化名称；
- 执行频率；
- 输入输出目录；
- 创建时间；
- 负责人；
- 停用方式。

不再需要时应及时停用或删除，避免旧任务继续读取新文件。

## 首次运行检查清单

- 是否在正确时间触发；
- 是否读取了正确文件；
- 是否覆盖历史结果；
- 是否请求了额外权限；
- 是否在没有输入时正确停止；
- 输出是否包含本次运行日期；
- 是否发生了未授权发送或修改。
