# 用 WorkBuddy 制作 PPT：从资料到可检查的演示文稿

> 验证状态：B 级来源核对。官方文档确认支持演示文稿场景；本文模板尚未完成本项目的完整人工实测。

## 先准备这些信息

- 演示对象和使用场合；
- 预计页数和演讲时长；
- 必须包含的数据与结论；
- 品牌色、Logo 或参考风格；
- 原始 Word、Excel、PDF 或 Markdown 资料。

## 可直接复制的任务描述

```text
请根据我提供的资料制作一份 PPT。

使用场景：【例如：10 分钟项目周会】
受众：【例如：部门负责人】
页数：【例如：8—10 页】

要求：
1. 先输出页面大纲，等我确认后再生成完整 PPT；
2. 每页只表达一个核心观点；
3. 重要数字必须来自提供的资料，并标注来源文件；
4. 不确定的结论标记“待确认”，不要编造；
5. 图表写清标题、单位、时间范围和数据口径；
6. 保留可编辑文本和图表；
7. 同时输出 PPT 文件和一份 Markdown 演讲提纲；
8. 不要修改原始资料；
9. 完成后列出生成文件及路径。
```

## 推荐流程

1. 先生成并确认故事线和页面大纲；
2. 再生成 PPT 初稿；
3. 检查数据和引用；
4. 调整排版与视觉层级；
5. 最后压缩文字并进行演讲时间测试。

## 人工检查清单

- 是否有无法从资料中找到来源的数字；
- 是否把推测写成确定结论；
- 是否存在文字溢出、字号过小或图表不可读；
- 页面顺序是否支撑最终结论；
- 导出的文件是否可以继续编辑。
