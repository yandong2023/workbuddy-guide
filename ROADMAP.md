# WorkBuddy Guide Roadmap

本项目的目标不是追求文章数量，而是逐步成为普通用户真正能照着完成任务的 WorkBuddy 非官方中文手册。

## 当前阶段：内容基础与质量标准

已完成：

- 全场景内容地图；
- 官方与社区来源库；
- A/B/C 三级验证体系；
- Hermes 定时维护规则；
- 内容和敏感信息自动检查；
- 首批新手、办公、自动化和故障教程；
- 一批配有原创 SVG 的高质量场景教程。

## v0.2：把最高频任务做深

目标：完成首批 30 篇 P0 教程，并把重点教程升级到统一图文标准。

优先主题：

- 多文档合并与 Word 正式文档；
- Excel 数据清洗、图表和经营分析；
- 录音转会议纪要与行动项；
- 行业研究、用户反馈分析和事实核查；
- 小红书、短视频和产品内容多平台改写；
- 每周工作总结、远程下任务和自动化失败恢复；
- 安全使用总指南、敏感数据脱敏和连接器授权撤销。

完成标准：

- 每篇至少一个官方来源；
- 每篇包含可复制指令和结果验收；
- 普通用户教程至少两张有效配图；
- 高风险操作有明确安全边界；
- 不自动合并 AI 生成内容。

## v0.3：真实实测与样例文件

目标：让高流量教程从 B 级来源核对升级为 A 级实测。

计划：

- 建立无敏感信息的演示文件集；
- 在 Windows 和 macOS 上分别测试核心流程；
- 记录 WorkBuddy 版本、模型、Skill 和执行时间；
- 保存脱敏截图、失败过程和修正步骤；
- 提供结果文件结构示例；
- 每季度升级最多 5 篇，避免为了数字降低质量。

## v0.4：社区贡献体验

目标：让普通用户也能在 10—20 分钟内完成一次贡献。

计划：

- 完善来源推荐、教程需求和实测报告模板；
- 建立“第一次贡献”指南；
- 发布适合新贡献者的小任务；
- 公开贡献者名单和感谢记录；
- 允许用户只提交错误截图、失效步骤或结果反馈，不要求会写代码。

## v0.5：提示词和检查清单库

目标：用户即使不读完整教程，也能快速复制一个安全模板开始工作。

计划：

- 文档、数据、调研、内容和自动化提示词；
- 执行前、执行中、执行后检查清单；
- 文件处理、数据分析、发布和权限风险模板；
- 按角色整理常用工作流入口。

## v0.6：文档站与搜索

触发条件：内容和维护流程稳定，而不是只看 Star 数量。

计划：

- 使用 GitHub Pages、VitePress 或同类静态文档方案；
- 支持全文搜索、侧边栏、标签和更新时间；
- 保持 Markdown 为唯一内容源；
- 以后可绑定独立域名；
- GitHub 仓库继续作为贡献、版本和自动化中心。

## v1.0：普通人的 AI 工作手册

长期方向：

- WorkBuddy 全场景中文教程；
- 按岗位组织的工作流合集；
- 可复制提示词和安全检查清单；
- 真实实测和版本更新记录；
- 社区纠错和贡献机制；
- 必要时扩展到更广泛的 AI Agent 办公方法，但保持 WorkBuddy 为核心入口。

## 我们如何衡量项目是否变好

Star 是重要的传播信号，但不是唯一指标。更重要的是：

- 有多少教程被真实使用；
- 是否有人提交纠错和实测反馈；
- 高访问教程是否保持更新；
- 用户是否能按照步骤完成任务；
- 是否减少了文件误删、虚构内容和隐私泄露风险；
- 是否出现稳定的外部贡献者。

完整内容状态见 [`content/coverage-matrix.yaml`](content/coverage-matrix.yaml)。
