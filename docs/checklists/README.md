# WorkBuddy 执行检查清单

这些清单适合在开始任务、批准执行、验收结果和设置自动化时使用。

## 一、开始任务前

- [ ] 我使用的是文件副本，不是唯一一份原文件
- [ ] 已建立独立工作目录
- [ ] `input/`、`work/`、`output/` 和 `logs/` 分开
- [ ] 已移除与任务无关的客户、员工和个人信息
- [ ] 知道当前使用的模型、Skill 和连接器
- [ ] 已确认文件是否可能发送到第三方服务
- [ ] 默认使用默认权限
- [ ] 新任务先使用 Ask 或 Plan
- [ ] 已明确禁止删除、覆盖、发送和发布
- [ ] 已指定输出文件名和保存位置
- [ ] 已说明不确定信息必须标记“待确认”

## 二、批准 Plan 前

- [ ] 计划中的输入范围正确
- [ ] 没有访问整个桌面、下载目录或磁盘
- [ ] 没有修改 `input/` 中的原文件副本
- [ ] 生成了清单或预览
- [ ] 复杂任务先做小样测试
- [ ] 大文件任务已经分批
- [ ] 批次有进度和恢复记录
- [ ] 使用脚本或第三方工具前会询问
- [ ] 发送对象、发布平台和公开范围需要人工确认
- [ ] 失败时会停止，而不是无限重试

## 三、任务完成后

- [ ] 实际文件已经生成，而不只是对话中显示结果
- [ ] 输出文件可以正常打开
- [ ] 原文件保持不变
- [ ] 文件名、格式和保存路径正确
- [ ] 数字、页码、时间戳和来源可以抽查
- [ ] 没有把推测写成确定事实
- [ ] 没有遗漏失败批次
- [ ] WorkBuddy 变更列表和磁盘实际情况一致
- [ ] 临时文件和脚本用途明确
- [ ] 不再需要的完全访问或连接器授权已经撤销

## 四、数据和 Excel

- [ ] 字段含义已经确认
- [ ] 日期、金额、百分比和编号类型正确
- [ ] 清洗规则有记录
- [ ] 重复数据先预览再处理
- [ ] 合并后保留来源文件和工作表
- [ ] 合并前后行数可以对账
- [ ] 核心指标可以抽样复算
- [ ] 图表包含标题、单位、时间和口径
- [ ] 异常和缺失数据没有被隐瞒

## 五、调研和研究报告

- [ ] 关键事实有原始链接
- [ ] 记录发布日期、访问日期和数据年份
- [ ] 官方、一手资料优先
- [ ] 媒体观点和用户反馈与官方事实分开
- [ ] 冲突信息并列展示
- [ ] 搜索摘要没有替代原文
- [ ] 推断明确标记为推断
- [ ] 价格、功能、政策和模型标注最后核对日期
- [ ] 结论能够追溯到证据表

## 六、内容和营销

- [ ] 没有虚构客户、收入、案例和效果数字
- [ ] 产品功能和价格已核对
- [ ] 引用和图片版权已检查
- [ ] 不使用未经授权的第三方文章、音乐和视频
- [ ] 标题和宣传语不过度夸大
- [ ] 平台规则和广告法风险已人工检查
- [ ] 只生成草稿和预览
- [ ] 上传、发送和发布由人完成

## 七、自动化

- [ ] 手动执行至少两次并通过验收
- [ ] 输入目录和命名规则固定
- [ ] 输出文件包含日期或批次，不覆盖历史结果
- [ ] 没有输入时会停止并说明
- [ ] 失败时有限次重试
- [ ] 不自动删除中间文件
- [ ] 不自动付款、发送或公开发布
- [ ] 有明确的暂停和关闭方式
- [ ] 定期检查自动化是否仍然需要
- [ ] 记录负责人和最后检查日期

## 八、公开提交到 GitHub 前

- [ ] 没有 `.env`、API Key、Token、Cookie 和密码
- [ ] 没有 OAuth 链接、二维码和认证目录
- [ ] 没有真实用户名和本地路径
- [ ] 没有客户、订单、合同和财务数据
- [ ] 截图已经脱敏
- [ ] 第三方内容没有大段复制
- [ ] 来源和许可证已经注明
- [ ] 来源核对没有写成亲测
- [ ] 已运行 `python scripts/validate_content.py`

详细安全说明见仓库根目录的 [`SECURITY.md`](../../SECURITY.md)。
