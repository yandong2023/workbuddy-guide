# 第一次贡献：不会写代码也可以参与

这个项目最需要的不只是代码，也需要真实用户反馈、教程纠错、来源推荐和脱敏截图。

## 最容易的四种贡献

### 1. 报告教程错误

适合发现按钮变化、步骤失效、链接打不开或结果与教程不同的用户。

打开仓库 Issues，选择“教程纠错”，填写：

- 对应教程；
- 操作系统和 WorkBuddy 版本；
- 实际看到的情况；
- 错误提示；
- 可核对来源；
- 脱敏截图。

不需要自己修改 Markdown。

### 2. 推荐公开教程或资料

选择“推荐教程或资料来源”，提交：

- 标题和链接；
- 作者、发布者和日期；
- 适合补充哪个工作场景；
- 资料中最有价值的步骤或失败案例；
- 哪些说法可能已经过期。

只提供链接和简短说明，不要复制全文或完整视频字幕。

### 3. 提交实测反馈

选择“提交实测结果”，记录：

- Windows 或 macOS；
- WorkBuddy 版本；
- 使用模式、模型和 Skill；
- 哪些步骤成功；
- 哪些步骤失败；
- 与教程有什么差异；
- 是否愿意提供脱敏截图。

一份真实失败记录，往往比十篇自动生成文章更有价值。

### 4. 直接修改教程

适合愿意使用 GitHub 的贡献者：

1. Fork 仓库；
2. 从 `main` 创建分支；
3. 修改一个清晰主题；
4. 运行 `python scripts/validate_content.py`；
5. 创建 Pull Request；
6. 说明来源、验证等级和待确认事项。

## 新教程的最低要求

- 解决一个明确任务；
- 至少一个官方来源；
- 说明是否完成实测；
- 提供可复制指令；
- 包含结果验收和常见错误；
- 写明撤销恢复方法；
- 说明隐私、权限和数据去向；
- 普通用户教程至少两张有效配图；
- 不搬运第三方全文和图片。

模板见：[`templates/tutorial-template.md`](../../templates/tutorial-template.md)。

## 适合第一次贡献的小任务

- 修正错别字或失效内链；
- 给现有教程补一条常见问题；
- 提供一个脱敏失败案例；
- 推荐一个官方文档或公开教程；
- 给教程补充最后核对日期；
- 绘制一张解释流程的原创 SVG；
- 将一个 B 级教程在真实环境中跑一遍并报告结果。

## 截图要求

提交截图前确认：

- 使用虚构或脱敏资料；
- 隐藏账号、邮箱、通知和本地用户名；
- 不显示 Token、Cookie、授权二维码和回调链接；
- 一张图只说明一个主要动作；
- 写明操作系统、截图日期和可确认的 WorkBuddy 版本。

## 贡献者会得到什么

有效贡献会保留在 Git 历史和 Pull Request 中。后续项目会建立贡献者名单，感谢提供教程、实测、纠错、配图和来源的参与者。

完整规则见仓库根目录的 [`CONTRIBUTING.md`](../../CONTRIBUTING.md) 和 [`CODE_OF_CONDUCT.md`](../../CODE_OF_CONDUCT.md)。
