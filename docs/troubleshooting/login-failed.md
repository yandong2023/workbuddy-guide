# WorkBuddy 点击登录没反应或登录失败怎么办

> 验证状态：B 级来源核对。本文依据官方中文 FAQ 和快速开始文档整理，尚未完成本项目的完整人工实测。登录方式可能随地区、版本和账号体系变化。

登录通常需要客户端拉起浏览器完成授权，再返回 WorkBuddy。任何一个环节被默认浏览器、网络、系统权限或安全软件拦截，都可能表现为“点击没反应”。

## 先确认登录方式

官方不同语言文档展示的登录方式可能不同，例如浏览器第三方授权或扫码。请以当前客户端实际显示的入口为准，不要使用陌生人提供的“免登录脚本”或修改认证文件。

## 第一步：检查默认浏览器

官方中文 FAQ 建议先确认系统已经设置可用的默认浏览器。

### Windows

进入：

```text
系统设置 → 应用 → 默认应用
```

确认 Chrome、Edge 或其他可用浏览器能够打开 `https` 链接。

### macOS

进入：

```text
系统设置 → 桌面与程序坞 → 默认网页浏览器
```

部分系统版本入口可能不同。

设置后完全退出 WorkBuddy，再重新打开并登录。

## 第二步：观察浏览器授权流程

点击登录后检查：

- 浏览器是否打开；
- 页面是否来自官方域名；
- 是否被浏览器拦截弹窗；
- 是否完成授权后自动返回客户端；
- 是否出现回调失败或网络错误。

如果客户端提供“复制链接”选项，可以把链接粘贴到可信的默认浏览器中打开。不要把授权链接发给别人。

## 第三步：检查网络和代理

- 官方网页是否能正常打开；
- 系统时间是否正确；
- VPN 或代理是否导致回调失败；
- 公司网络是否拦截登录域名；
- 浏览器和客户端是否走了不同网络；
- 是否可以临时换一个可信网络测试。

不要使用来源不明的代理配置或证书。

## 第四步：检查安全软件

官方 FAQ 记录了安全软件可能拦截客户端。测试时可以查看：

- Windows 安全中心的阻止记录；
- 防火墙提示；
- 第三方安全软件的隔离区；
- 是否拦截浏览器回调或客户端联网。

不建议为了登录长期关闭系统防护。应优先查看具体拦截记录，并仅对官方安装程序和客户端设置必要许可。

## 第五步：检查应用数据目录权限

官方 FAQ 提到，账号数据目录不属于当前系统用户时可能导致登录失败。修改目录所有者属于高风险系统操作：

- 先确认路径确实来自官方 FAQ；
- 先记录现有权限；
- 不要复制执行来源不明的管理员命令；
- 公司电脑应联系 IT 管理员；
- 不熟悉终端或 PowerShell 时，不要直接运行递归授权命令。

公开提问时不要粘贴完整用户名、目录列表或认证文件内容。

## 安全测试顺序

1. 设置默认浏览器；
2. 重新启动 WorkBuddy；
3. 检查官方授权页面；
4. 更换可信网络；
5. 查看安全软件拦截记录；
6. 检查当前系统用户权限；
7. 使用客户端反馈入口提交问题。

## 提交反馈前准备

- 操作系统和版本；
- WorkBuddy 版本；
- 登录入口类型；
- 默认浏览器；
- 点击后的实际现象；
- 完整错误文本；
- 是否使用代理；
- 是否在升级后出现；
- 已尝试的步骤。

不要提交 Cookie、Token、授权回调链接、二维码、账号密码或完整认证目录。
