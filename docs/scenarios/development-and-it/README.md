## P1：IT 与运维

- 服务器日志错误摘要；
- 配置文件差异对比；
- 发布说明和 Changelog；
- 测试用例生成与覆盖检查；
- SQL 分析和数据字典生成；
- API 文档和示例请求；
- 安装脚本和环境检查；
- 故障复盘报告；
- 资产清单和软件版本盘点；
- 安全扫描结果整理；
- GitHub Issue 和 PR 摘要；
- 定时检查构建或任务状态；
- 技术调研和方案对比；
- 从原型生成本地 HTML 工具。

## 安全要求

- 不把生产密钥、数据库密码、Cookie 和内部 Token 放入任务；
- 执行脚本、安装依赖、修改系统配置和访问网络前确认；
- 修改代码前创建分支并保留可回滚点；
- 生产部署、数据库迁移和删除资源不得全自动执行；
- 日志、错误栈和配置文件公开前必须脱敏；
- 自动修复后必须运行测试并审查变更。
