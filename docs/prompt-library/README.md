# WorkBuddy 可复制任务模板库

这些模板用于快速开始，不代表所有环境都能直接成功。第一次使用时建议选择 **Plan**，使用独立工作目录、原文件副本和默认权限。

每个模板都包含四个核心部分：输入、处理、输出和限制。请把方括号中的内容替换成自己的信息。

## 通用安全前缀

可以把这段放在任何文件任务前面：

```text
只处理当前工作目录。
input/ 中的文件只读，不得修改、移动、重命名、覆盖或删除。
所有中间文件保存到 work/，最终结果保存到 output/。
不确定的信息标记“待确认”，不要编造。
涉及联网、运行脚本、安装工具、访问工作目录之外的位置、发送或发布内容时，先说明并等待我确认。
完成后列出所有读取、新建、修改、移动和删除的文件。
```

# 一、文档与文件

## 1. 总结 PDF，并保留页码

```text
请阅读 input/report.pdf。

要求：
1. 先确认总页数、目录和可读取情况；
2. 用 300 字以内概括核心结论；
3. 按章节整理观点、数据、方法和限制；
4. 每个重要结论标注原始页码；
5. 区分原文陈述和你的推断；
6. 无法读取的页面写入 output/failed-pages.csv；
7. 输出 output/report-notes.md；
8. 不要修改原 PDF。
```

## 2. 对比多份 Word/PDF

```text
请对比 input/ 中的全部 Word 和 PDF 文件。

第一步只建立文件清单和版本关系，不要判断哪个版本正确。
随后按以下维度比较：
- 新增、删除和改写内容；
- 金额、日期、责任人、范围和条件；
- 否定词、例外条款和附件；
- 不同文件之间的冲突和遗漏。

输出：
- output/difference-matrix.xlsx；
- output/high-risk-conflicts.md；
- output/merge-suggestions.md。

每项差异保留来源文件和页码或章节位置。
```

## 3. 多文档合并成一份报告

```text
请把 input/ 中的多份资料合并成一份结构化报告。

要求：
1. 先生成资料清单和建议目录；
2. 相同内容去重；
3. 冲突信息并列展示，不擅自裁决；
4. 重要事实保留来源文件和页码；
5. 统一标题、术语、日期和格式；
6. 无法确认的内容标记“待确认”；
7. 输出 output/combined-report.md；
8. 另外输出 output/source-map.csv，记录每个章节使用了哪些来源。
```

## 4. 批量重命名文件

```text
请分析 input/files/ 中的文件，但暂时不要重命名。

命名规则：【例如：YYYY-MM-DD_项目_类型_序号.原扩展名】

先生成 output/rename-preview.csv，字段包括：
- 原路径；
- 原文件名；
- 建议新文件名；
- 判断依据；
- 同名冲突；
- 风险；
- 是否需要人工确认。

等我确认后，只处理 status=approved 的文件。
不得覆盖已有文件，必须保存原名和新名映射以便恢复。
```

# 二、Excel、数据与汇报

## 5. Excel 数据清洗

```text
请分析 input/data.xlsx。

第一步先列出工作表、字段、数据量和字段类型，不要修改文件。
随后检查：
1. 空值；
2. 重复数据；
3. 日期、金额、百分比和编号格式；
4. 明显异常值；
5. 字段含义不一致；
6. 公式错误。

输出：
- output/cleaned-data.xlsx；
- output/cleaning-log.csv；
- output/validation-report.md。

不要删除无法确认的数据，把它们标记为 needs-review。
```

## 6. 合并多份 Excel

```text
请合并 input/ 中的全部 Excel 文件。

流程：
1. 生成文件、工作表和字段清单；
2. 建立字段映射表，等我确认；
3. 先合并 3 个样本文件；
4. 检查数据类型、重复项和来源字段；
5. 再分批处理全部文件。

最终文件必须包含 source_file 和 source_sheet 两列。
输出：
- output/merged.xlsx；
- output/field-mapping.csv；
- output/duplicates-preview.csv；
- output/exceptions.csv；
- output/merge-report.md。
```

## 7. 销售经营分析

```text
请分析 input/sales.xlsx。

按月份、产品、地区、渠道和客户汇总：
- 销售额；
- 订单量；
- 客单价；
- 退款额；
- 毛利或已有利润字段；
- 同比和环比。

要求：
1. 先确认字段和指标口径；
2. 缺少必要字段时停止并说明；
3. 找出增长和下降最明显的部分；
4. 区分数据事实和原因推测；
5. 图表标明标题、单位、时间和口径；
6. 输出 output/sales-analysis.xlsx 和 output/management-summary.md。
```

## 8. 数据报告转 PPT

```text
请根据 input/report.xlsx 和 input/notes.md 制作汇报 PPT。

第一阶段只输出 8—10 页页面大纲，等我确认后再生成 PPT。

要求：
- 每页只有一个核心观点；
- 数字必须来自输入文件；
- 图表标注单位、时间和口径；
- 推断标记为推断；
- 保留可编辑文本和图表；
- 输出 output/report.pptx 和 output/speaker-notes.md；
- 不要修改原始数据。
```

# 三、会议、项目与管理

## 9. 会议纪要

```text
请根据 input/meeting-notes.txt 整理正式会议纪要。

要求：
1. 提取会议主题、日期、参与角色和议程；
2. 按议题整理讨论要点；
3. 单独列出已确认结论；
4. 单独列出争议和待确认问题；
5. 行动项包含事项、负责人、截止时间、依赖和状态；
6. 未明确的负责人和日期写“待确认”，不要猜测；
7. 输出 output/meeting-minutes.md 和 output/action-items.csv。
```

## 10. 项目计划拆解

```text
请根据 input/project-brief.md 制定项目计划。

先识别：目标、范围、非目标、交付物、截止时间、参与角色和已知约束。

输出：
1. 工作分解结构；
2. 阶段和里程碑；
3. 每项任务的负责人角色、依赖和验收标准；
4. 风险清单；
5. 待确认问题；
6. output/project-plan.xlsx；
7. output/project-summary.md。

不要替负责人承诺具体排期，无法确认时使用角色或“待确认”。
```

## 11. 周报和管理摘要

```text
请根据 input/ 中的工作记录、会议纪要和项目数据生成周报。

按以下结构组织：
- 本周目标；
- 完成事项和结果；
- 数据变化；
- 未完成事项及原因；
- 风险和依赖；
- 下周计划；
- 需要支持。

不要把计划写成已经完成，不要编造数据。
输出 500 字简版和详细版到 output/weekly-report.md。
```

# 四、调研、产品与决策

## 12. 竞品调研

```text
我要研究【产品/公司】的竞品，用于【定价/立项/功能规划/采购】。

第一步先提出需要确认的研究范围和比较维度。
随后搜集：
- 官方网站和定价；
- 官方文档和产品演示；
- 公开评测；
- 用户反馈和常见问题。

要求：
1. 关键事实保留原始链接、发布日期和访问日期；
2. 官方说法、第三方观点和用户反馈分开；
3. 价格写明币种和周期；
4. 冲突信息并列展示；
5. 推断明确标记为推断；
6. 输出 output/evidence.csv、output/competitor-matrix.xlsx 和 output/report.md。
```

## 13. 用户反馈转需求

```text
请分析 input/feedback/ 中的反馈资料。

第一步先检查并提示需要脱敏的信息。
随后：
1. 标准化渠道、日期、用户类型、场景、原话和问题；
2. 对重复反馈生成预览，不直接删除；
3. 按用户目标和问题聚类；
4. 统计反馈数和独立用户数；
5. 保留代表性原话和来源；
6. 生成需求候选、证据强度、影响范围和验证方式；
7. 不替产品负责人决定优先级和排期。

输出 output/feedback-normalized.csv、output/problem-clusters.md 和 output/requirement-candidates.xlsx。
```

## 14. 行业研究报告

```text
请研究【行业】在【地区】的现状，用于【决策目的】。

研究范围：
- 市场定义和边界；
- 主要参与者；
- 商业模式；
- 用户和需求；
- 产业链；
- 关键趋势；
- 风险和不确定性。

要求：
1. 优先官方和一手来源；
2. 每个数字记录年份、币种、口径和来源；
3. 不同口径不能直接相加；
4. 搜索摘要不能替代原文；
5. 事实、推断和建议分开；
6. 输出证据表、研究报告和一页管理摘要。
```

# 五、内容、营销与自动化

## 15. 公众号文章

```text
请根据 input/source-materials/ 生成一篇公众号文章草稿。

流程：
1. 先建立事实卡片，列出可用数据、案例和来源；
2. 给出 5 个选题和目标读者；
3. 生成大纲，等我确认；
4. 再写正文；
5. 检查事实、版权、广告法风险和夸张表达；
6. 生成图片 Brief；
7. 输出 Markdown 和富文本 HTML 草稿。

禁止虚构客户、收入、效果和用户评价。
不要登录公众号后台，不要上传或发布。
```

## 16. 短视频脚本和分镜

```text
请根据 input/materials.md 生成一条【60 秒】短视频脚本。

要求：
- 前 3 秒说明用户痛点或结果；
- 脚本包含画面、口播、字幕和时长；
- 每个事实有来源；
- 不夸大效果；
- 给出 3 个不同钩子；
- 生成分镜表和拍摄素材清单；
- 输出 output/script.md 和 output/storyboard.csv；
- 不自动发布。
```

## 17. 产品资料拆成多平台内容

```text
请把 input/product-materials/ 中的资料改写成：
1. 公众号文章大纲；
2. 小红书图文；
3. 朋友圈文案；
4. 60 秒短视频口播；
5. 社群介绍；
6. FAQ。

所有版本必须使用同一份事实卡片，但根据平台调整结构和长度。
不得虚构功能、价格、客户案例和效果数字。
输出到 output/，每个平台单独一个文件。
```

## 18. 每日资讯简报

```text
请生成今天的资讯简报。

时间范围：过去 24 小时。
关注主题：【填写主题】。

要求：
1. 每条包含标题、摘要、原始来源和发布日期；
2. 同一事件去重；
3. 官方、一手来源优先；
4. 旧闻不要当作新消息；
5. 每条说明为什么值得关注；
6. 没有重要更新时允许写“无重要更新”；
7. 保存为 output/YYYY-MM-DD-briefing.md；
8. 不自动发送。
```

## 19. 安全的定时周报自动化

```text
每周【周五 17:00】生成工作总结草稿。

输入：只读取当前工作目录 input/ 中本周新增的工作记录。
输出：output/YYYY-WW-weekly-report.md。

规则：
- 没有新记录时生成“无可用输入”说明；
- 不使用旧数据冒充本周数据；
- 不覆盖历史文件；
- 不发送邮件或群消息；
- 失败时记录错误，不无限重试；
- 完成后通知我人工审核。
```

## 20. 自动化失败后的恢复

```text
请检查本次自动化任务的运行记录和输出目录。

要求：
1. 列出已完成、失败、重复和未开始的步骤；
2. 检查是否产生半成品或重复文件；
3. 不覆盖现有结果；
4. 生成恢复计划，说明从哪里继续；
5. 只重试失败或未完成部分；
6. 在我确认计划前不要执行；
7. 输出 logs/recovery-plan.md。
```

## 使用模板后的验收问题

任何任务结束后，都可以问：

```text
请给我一份验收报告：
1. 实际读取了哪些文件；
2. 新建、修改、移动和删除了哪些文件；
3. 哪些步骤成功、失败或未执行；
4. 哪些事实、数字或结论仍需人工核对；
5. 是否访问网络或第三方服务；
6. 是否存在隐私、权限或覆盖风险；
7. 下一步建议是什么。
```
