# 管理、战略与团队领导

适合团队负责人、部门经理、创业者和需要做经营决策的人。

## P1 内容计划

| 主题 | 状态 | 重点内容 | 计划路径 |
|---|---|---|---|
| 部门周报/月报汇总 | 🔵 周报和数据来源可复用 | 目标、结果、问题、风险和支持 | `department-report.md` |
| OKR 草稿和检查 | ⚪ 待研究 | 目标、关键结果、可衡量性、依赖 | `okr-draft.md` |
| 经营复盘 | 🔵 数据和复盘来源可复用 | 目标、实际、差异、原因、行动 | `business-review.md` |
| 决策备忘录 | ⚪ 待研究 | 问题、方案、证据、风险和建议 | `decision-memo.md` |
| 管理会议材料 | 🔵 文档和 PPT 能力可复用 | 议题、数据、决策点和预读 | `management-meeting.md` |
| 一对一会议准备 | ⚪ 待研究 | 进展、阻塞、反馈、发展和行动项 | `one-on-one.md` |
| 团队工作负载分析 | ⚪ 待研究 | 任务、时间、优先级、风险和公平性 | `workload-analysis.md` |
| 团队问题和风险汇总 | ⚪ 待研究 | 来源、频次、影响、负责人和趋势 | `team-risk-summary.md` |
| 战略资料整理 | 🔵 调研来源可复用 | 市场、竞争、能力、假设和选择 | `strategy-research.md` |
| 年度计划草稿 | ⚪ 待研究 | 目标、举措、资源、里程碑和风险 | `annual-plan.md` |

## 其他场景

- 管理层简报；
- 跨部门问题清单；
- 资源需求汇总；
- 组织沟通稿；
- 目标拆解到团队计划；
- 会议决策跟踪；
- 项目组合优先级；
- 部门预算说明；
- 团队能力和培训需求；
- 业务异常周报；
- 领导讲话和汇报稿草稿；
- 公司知识和历史决策整理。

## 强制标准

- 不把 AI 判断作为招聘、绩效、晋升、薪酬和裁员的唯一依据；
- 管理报告中的数据和评价必须可追溯；
- 推测、风险和事实分开；
- 涉及员工的材料应最小化使用个人信息；
- 决策备忘录必须列出反对意见和不确定性；
- 正式沟通和组织政策发布前必须人工审批。

## 每篇教程需要的配图

- 目标到关键结果和行动的结构；
- 经营复盘闭环；
- 方案、证据、风险和决策矩阵；
- 团队工作负载和优先级示意；
- “AI 整理 → 管理者判断 → 组织执行”的责任边界。
