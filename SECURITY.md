# 安全政策

本仓库是公开的教程和自动化项目，不应包含任何真实密钥、认证信息、客户数据或未脱敏业务资料。

## 报告安全问题

发现以下问题时，请不要在公开 Issue 中粘贴敏感内容：

- API Key、Token、Cookie、OAuth 回调或二维码泄露；
- 客户、员工、患者、订单、合同或财务信息泄露；
- 教程可能导致文件误删、覆盖或未经确认发送；
- Skill、脚本或连接器存在可疑联网或越权行为；
- 仓库历史中出现已删除但仍可访问的秘密信息；
- 自动化工作流可能直接发布、付款或修改重要账号。

请通过 GitHub 仓库所有者的公开联系方式进行私下报告，只提供能够定位问题的最少信息。不要发送真实密钥和完整敏感文件；可以使用脱敏样例说明。

## 处理原则

收到有效报告后，维护者将优先：

1. 暂停或标记受影响的教程、Skill 或自动化；
2. 删除公开内容并在必要时清理 Git 历史；
3. 提醒相关人员立即轮换密钥和撤销授权；
4. 核对影响范围；
5. 修复教程和自动检查规则；
6. 在不泄露安全细节的前提下记录修复结果。

## 公共仓库禁止提交

- `.env` 和任何密钥文件；
- API Key、Token、Cookie 和密码；
- OAuth 授权链接、二维码和回调参数；
- 原始运行日志、浏览器配置和认证目录；
- 客户姓名、手机号、邮箱、订单号和合同正文；
- 公司生产数据库、服务器和内部网络信息；
- 未脱敏截图、录音、表格和文档；
- 来自付费课程或受版权保护内容的全文。

## WorkBuddy 教程的安全边界

本项目中的教程默认遵守：

- 本地文件不等于完全离线；
- 原始文件使用副本；
- 默认权限优先于完全访问；
- 删除、覆盖、移动、发送、发布和支付需要人工确认；
- 第三方 Skill 和连接器在使用前检查作者、权限、联网和数据去向；
- 高风险岗位决策由负责人完成，AI 只做辅助整理。

安全问题优先级高于内容更新和发布速度。
