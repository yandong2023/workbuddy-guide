## P1：协作场景

- 会议议程和会前材料准备；
- 根据历史纪要生成本次会议问题清单；
- 会议中的决策记录和版本管理；
- 每日站会内容汇总；
- 项目需求变更影响分析；
- 需求文档拆成研发任务；
- 甘特图或里程碑表草稿；
- 任务延期原因分类；
- 跨部门依赖跟踪；
- 项目关闭和交付清单；
- 客户项目阶段汇报；
- 项目文档目录和知识移交。

## 每篇教程需要的配图

- 会议输入到纪要、待办、提醒的流程图；
- 会议纪要标准结构示意；
- 行动项表格示例；
- 项目计划层级图；
- 状态看板或进度汇总示意。

## 安全要求

- 未明确的负责人和截止时间不得自动猜测；
- 会议中的建议不能自动写成团队结论；
- 对外发送、创建日历事件和通知成员前必须确认；
- 会议录音和内部决策资料应遵守组织的数据政策。
