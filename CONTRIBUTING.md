# 贡献指南

感谢你帮助完善 WorkBuddy 非官方中文使用手册。

## 可以贡献什么

- 新手教程、办公案例、自动化工作流和 Skill 教程；
- 已有教程的版本更新；
- 错别字、失效链接、错误步骤和安全风险修复；
- 真实执行结果、失败原因和脱敏截图；
- 可靠来源和用户高频问题。

## 提交前要求

1. 优先使用官方文档和官方更新作为事实来源；
2. 第三方内容只能用于研究和交叉核对，不得大段复制；
3. 不得把“来源核对”描述为“已实测”；
4. 不得虚构按钮、路径、版本、截图、积分消耗或执行结果；
5. 每篇非目录教程必须使用 `templates/tutorial-template.md` 的元数据结构；
6. 每篇 B 级教程至少提供一个官方来源；
7. A 级教程必须写明 `verified_at`、测试平台和必要的版本信息；
8. 涉及删除、覆盖、发送、支付、账号、隐私或完全访问权限时，必须加入风险提示；
9. 自动生成内容只能通过 PR 提交，不能直接推送到 `main`。

## 验证等级

- `tested`：A 级，完整人工实测；
- `source-verified`：B 级，可靠来源核对；
- `automated-draft`：C 级，自动草稿，只能位于 `drafts/` 或 PR 中。

## 推荐流程

1. 创建 Issue，说明用户问题和来源；
2. 新建分支；
3. 建立研究证据包；
4. 编写或修改教程；
5. 运行 `python scripts/validate_content.py`；
6. 创建 Pull Request；
7. 完成人工审核后再合并。

## 截图规范

- 仅使用自己拍摄、官方允许使用或明确授权的截图；
- 隐去姓名、邮箱、Token、文件路径中的个人信息和业务数据；
- 图片放在 `assets/screenshots/<文章 slug>/`；
- 图片文件名使用小写英文和连字符。

## 公共仓库安全红线

严禁提交密钥、Cookie、完整个人画像、客户身份、订单号、生产数据库信息、原始日志和未脱敏文件。发现泄露后应立即删除相关提交并轮换密钥。
